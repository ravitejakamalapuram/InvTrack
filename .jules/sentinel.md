## 2024-05-24 - [Secure Storage] Enable encryptedSharedPreferences for FlutterSecureStorage on Android
**Vulnerability:** The `FlutterSecureStorage` plugin was being used on Android with default options (`const AndroidOptions()`). In older configurations, this defaults to using standard, unencrypted `SharedPreferences` for storing the encrypted payload, with only the key stored in the KeyStore.
**Learning:** For modern Android security standards, it is crucial to explicitly enable `encryptedSharedPreferences: true` when configuring `FlutterSecureStorage`. This ensures that both the keys and the values are encrypted using Android's native `EncryptedSharedPreferences`, providing an additional layer of defense against local extraction.
**Prevention:** Always verify that `FlutterSecureStorage` configurations explicitly set `encryptedSharedPreferences: true` in `AndroidOptions` for new applications or features. Note: Migrating an existing app requires careful handling to transfer data from the old unencrypted preferences to the new encrypted ones.
## 2024-05-24 - [Local Storage] Prevent IDOR and Path Traversal in Document Storage
**Vulnerability:** The `DocumentStorageService` checked if file paths were safe by verifying they fell within the global `getApplicationDocumentsDirectory()`. This would have allowed users to access other users' files on the same device (Cross-Account Local Data Access/IDOR), because multiple users share the same global application document directory if they sign in on the same device.
**Learning:** File path validations for user-specific data must be strictly scoped to the user's isolated directory (`_documentsDirectory`), not the global application directory. Checking against the global application directory defeats the purpose of isolating data by user ID in local storage.
**Prevention:** When implementing `_isSafePath` or similar path validation methods for user-specific documents, always validate the `canonicalPath` against the specific user's `canonicalized` sub-directory (`_documentsDirectory`). Never use global application directories for validation when data is segmented by user account.
